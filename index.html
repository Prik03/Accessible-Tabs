Manual Activation Tablist

  • role="tablist" is used on the parent wrapper to notify assistive technology that it contains multiple tabs.
  • It groups all tabs together and provides context like: "This is a tablist with n tabs."

  • Each tab uses role="tab" so screen readers know these are selectable items.
  • Only one tab is active at a time â†’ marked with aria-selected="true" and tabindex="0".
  • Inactive tabs have aria-selected="false" and tabindex="-1" to prevent accidental focus.
  • aria-controls="tabpanel_x" links each tab to the panel it controls.

  • role="tabpanel" is used on the content panels associated with each tab.
  • aria-labelledby="tab_x" connects the panel back to its tab.
  • Hidden panels use the hidden attribute but remain available for screen readers when active.

role="presentation" tells screen readers to ignore the element because it is only used for layout or design.

HTML Code Structure


<div role="tablist" aria-label="Sample Tabs">
<button role="tab" aria-selected="true" aria-controls="tabpanel_1" tabindex="0">First Tab</button>
<button role="tab" aria-selected="false" aria-controls="tabpanel_2" tabindex="-1">Second Tab</button>
<button role="tab" aria-selected="false" aria-controls="tabpanel_3" tabindex="-1">Third Tab</button>
<button role="tab" aria-selected="false" aria-controls="tabpanel_4" tabindex="-1">Fourth Tab</button>
</div>

<div role="tabpanel" id="tabpanel_1" aria-labelledby="tab_1">Content for the first tab</div>
<div role="tabpanel" id="tabpanel_2" hidden>Content for the second tab</div>
<div role="tabpanel" id="tabpanel_3" hidden>Content for the third tab</div>
<div role="tabpanel" id="tabpanel_4" hidden>Content for the fourth tab</div>
